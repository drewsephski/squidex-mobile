# Release Notes 5.2.0

## Store / App Review Notes

- Added Cursor support alongside Codex and OpenCode.
- Bundled the Cursor app-server into the `clawdex-mobile` package.
- Improved `clawdex init` setup for multi-engine bridge installs.
- Added clearer Cursor API key setup guidance during onboarding.
- Added bridge-driven mobile UI surfaces for richer runtime details.

## Detailed Notes

### Cursor Support

- Added Cursor as a selectable bridge engine.
- Bundled the `cursor-app-server` command so Cursor setup does not require a separate package install.
- `clawdex init --engine cursor` now resolves the bundled Cursor app-server and saves the selected setup in `.env.secure`.
- Cursor setup now explains where to create the required Cursor account API key.

### Setup Reliability

- Hardened the published-package path so it uses bundled bridge binaries and bundled Cursor assets instead of source-checkout workspace assumptions.
- Increased bridge onboarding timeout to reduce false failures on slower first starts.
- Removed runtime setup fallbacks that could hide missing Cursor credentials or broken app-server paths.

### Mobile / Bridge UI

- Added bridge-driven UI payload support so the bridge can send richer details to the mobile app without requiring a provider-specific app release for every setup surface.
